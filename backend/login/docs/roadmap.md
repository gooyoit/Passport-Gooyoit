# Passport Roadmap

## v1.0 MVP

目标：先完成可上线接入的统一认证、SSO、用户和 RBAC 基础能力，让内部系统可以开始迁移到 Passport。

v1.0 包含：

- FastAPI 后端服务。
- MySQL 数据模型与 Alembic 迁移。
- Redis SSO Session 支撑。
- OAuth2 Authorization Code 风格登录流程。
- 邮箱验证码登录。
- 微信、Google、GitHub OAuth 登录接入点。
- 接入系统管理。
- 用户管理。
- 基于 PyCasbin 的 RBAC 角色权限管理。
- 默认角色自动继承策略。
- 中文管理后台。
- 中文统一登录页。

v1.0 不包含：

- 移动端 SDK。
- 复杂 ABAC 权限策略。
- 跨业务系统业务数据同步。
- 业务系统本地 Session 实现。

v1.0 验收标准：

- 至少 1 个业务系统可通过 Authorization Code 流程接入。
- 用户可通过邮箱验证码完成登录。
- 用户登录后可获得业务系统可验证的 Token。
- 公共用户池和私有用户池行为符合配置。
- 默认角色不产生 `user_roles` 冗余数据。
- 业务系统可查询用户角色和权限。
- 管理后台可完成接入系统、用户、角色、权限、登录方式的基本管理。

## v1.1 稳定性与运营能力

目标：补齐生产运行所需的可观测、审计、安全和运维能力。

计划能力：

- 登录、验证码、Token、管理后台接口限流。
- 审计日志查询和筛选。
- 登录历史查询。
- SSO Session 列表、强制下线、批量失效。
- `client_secret` 轮换流程。
- Refresh Token 撤销和设备维度管理。
- 邮件模板管理。
- 关键指标监控：登录成功率、验证码发送量、OAuth 回调失败率、Token 换取失败率。
- 管理后台操作确认和危险操作二次确认。

## v1.2 接入体验与多系统推广

目标：降低业务系统接入成本，支持更多内部系统批量迁移。

计划能力：

- 接入系统自助创建和配置向导。
- 接入文档和示例项目。
- 后端 SDK 或轻量客户端封装，优先支持 Python。
- 回调地址、登录方式、权限配置的配置检查。
- 系统级导入导出角色和权限。
- 批量邀请或导入系统用户。
- 管理后台内置接入调试工具。

## v1.3 第三方登录增强

目标：完善第三方身份绑定、解绑和账号冲突处理。

计划能力：

- 微信、Google、GitHub 登录配置后台化。
- 用户主动绑定和解绑第三方身份。
- 同邮箱多身份来源合并策略。
- 第三方登录失败原因可视化。
- OAuth `state` 和回调异常审计。
- 第三方头像、昵称同步策略。

## v2.0 高级权限与多端能力

目标：在 v1 系统稳定后，再引入更复杂的权限和终端形态。

计划能力：

- 移动端 SDK 或移动端登录适配。
- 更细粒度的授权模型评估，例如 ABAC 或条件权限。
- 多租户组织结构支持。
- SCIM 或企业目录同步。
- WebAuthn、Passkey 或 MFA。
- 更完善的单点登出协议。
- 跨区域部署和高可用架构。

## 暂不规划

以下能力不进入近期 Roadmap，除非后续业务明确需要：

- 替业务系统实现本地 Session。
- 同步业务系统私有业务数据。
- 在 Passport 中承载业务系统页面级菜单配置。
- 为所有语言一次性提供 SDK。
